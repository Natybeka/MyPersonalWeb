Text Editors vs IDEs when to use which

Connect with me:

        
By Natnael Bekabtu on March 7, 2020
----------------------------------------------------------------------------------------------------------------------------------------------

Most programmers use either IDEs or text editors to write, run or edit their code. Although there
is much debate as to what to use, both have some advantages over the other. IDEs or Integrated development
environments are softwares that have many facilities in building and running code. They usually have code
sensing tools, as well as comprehensive debuggers, compilers and testers. Some common IDE's include intellij,
Pycharm, Eclipse and Visual Studio. Although IDEs have a lot of tools built into them the downside is that they may
require a large amount of resource to run smoothly. It is not advised to use ides to edit 2 or 3 code files. IDEs are
mostly recommended when dealing with a huge project that has a lot of code files.

Text editors like sublime text, notepad++ or visual studio code give the simplicity that IDEs simply cannot provide.
Using editors in many ways makes amateur programmers better at their craft because each line of code has to be completed
manually instead of autocompleted. The programs can then be run from a command prompt using the necessary commands.
Nowadays text editors like visual studio code have plugins like code completion that can be downloaded for specific
programming language so that developers can install packages that can make their work easier. This gives the developer
more options as to what he/she requires when coding.
