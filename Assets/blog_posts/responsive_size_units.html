Responsive size units for Responsive web design

By Natnael Bekabtu on March 18, 2020

Before the advent of relative size units, pixel units were largely used to size items in the viewport (available window). px values are exactly intrepreted by the browser as they are. pixel is an absolute size unit meaning that it doesn't take into consideration the scalability of the web design when it is used across different devices with different screen sizes and resolutions, not to mention the need for the user to use the zoom feature of a browser for better visibility. Other absolute units include cm, mm and pt. Relative length units on the other hand specify a length relative to another property. Examples of relative units include em, % and rem. Each has its own advantage and drawback.

A responsive web design is one that involves the use CSS and HTML to automatically in order to make a website accessible to various devices (desktop, phones and tablets). Percent value units for length measure the length in proportion to the parent container. For example, if the there is a div tag inside the body tag and the its width attribute is set to 50%, then the browser translates it to "half the length of the body tag". If the parent container doesn't have a specified length attribute value then it goes up the browser tries to find the next parent in DOM tree heirarchy (in the example above the body tag) and take the corresponding relative measurement.

Percentage units are commonly used to specify the width and height of containers and to create responsive images. If the width property of an image is set to 100%, the image will be responsive and scales nicely for various devices A better option in many may be to use the "max-width" or "min-width" attribute, which makes the image scaled up of down if the viewport is larger or smaller than its original size.

Other major units are the rem and em units. The em is a unit that uses the font size of the styled (not the font-size of the parent element) element to translate it into the standard px value. This translation is done by the browser. When using em units, the pixel value is calculated by multiplying the font-size of the element being styled. For example, if a div has a font-size of 19px, a "padding value" of 10em is equivalent to 190px. The problem with using em arises when elements inherit font sizes from parent containers. Suppose we have a page with default font size (usually 16px) and a child div inside it with a padding of 1.5em, that div will inherit the font-size of 16px from the root element. This means that the padding is equivalent to 24px. If another div is then wrapped around the first div and set its font-size to 1.25em, it will inherit the root font-size of 16px and multiply it by 1.25 to get 20px. Adding a padding value of 1.5em would now equate to 30px. If wee keep adding div tags to this heirarchy with em based font-sizes the scaling effect for the font-size and padding attributes will be compounded, which will greatly hinder the visibility of the website. This is when using the rem unit would come in handy. The rem unit uses the font-size of the parent element rather than the styled element.

These three units can be used in conjunction to create a responsive website. Although web developers may not exactly agree on which unit to use where, most recommend some guidelines to maintain responsiveness. The percentage unit is optimal for image and div sizing and boosts their scalability. It is also not conventional to use em unit for font-size attributes because the resulting typography attributes like the padding and margin will not be proportional. Instead rem is recommended for setting the font-size attribute because it inherts attributes directly from the parent element. This gives more consistent resizing and ensures whatever default size a user's browser may have, the layout will adjust accordingly.
